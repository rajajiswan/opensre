## Create an account

1. **Sign up** at [app.tracer.cloud](http://app.tracer.cloud)
2. **Create a workspace**.

   <Frame>
     ![Create workspace](/images/createworkspace.png)
   </Frame>

## Connect Tracer to Slack

3. To start seeing incident investigations, you’ll need to connect Tracer to Slack. **Sign in with Slack** to start setting up the connection. 

   <Frame>
     <img
       src="/images/sign_in_with_slack(1).png"
       alt="Sign In With Slack(1)"
       title="Sign In With Slack(1)"
     />
   </Frame>
4. In Slack, pick your workspace and click **Allow.**

   <Frame>
     ![Give Slack Access](/images/give_slack_access.png)
   </Frame>
5. In Slack, open your alerts channel. 
6. Type /add and click **Add apps to this channel**
7. Search **Tracer** and click **Add**
8. In Tracer, open **Integrations** → **Slack**. Under **Channel Mappings** click **\+Add.** Select the channels you’ve added Tracer to, and select which types of alerts you’d like Tracer to respond to.

   <Frame>
     ![Channel Mapping Slack](/images/channel_mapping_slack.png)
   </Frame>

## Connect integrations

Each integration follows a simple, guided flow. Let’s look at Datadog as an example. 

1. Name - Helps distinguish between different instances
2. API Key - in Datadog, create a new API key and copy and paste into Tracer. 
3. Application Key - In Datadog, create a new Application Key and copy and paste into Tracer.

     ![Connect Datadog](/images/connect_datadog.png)
   </Frame>
